# 五、核心实现规则（开发必须遵守）

## 5\.1 静态打包规则

1. Vite构建模式设置为纯静态模式，禁用服务端渲染、禁用接口代理

2. 所有资源使用相对路径引用，禁止绝对路径、禁止跨域接口调用

3. 打包产物仅保留dist目录，无配置文件、无源码文件，可直接部署

4. WASM文件、下载资源文件直接放入public目录，构建时不编译、不压缩，保留原文件

## 5\.2 WebGPU实现规则

1. 直接使用浏览器原生navigator\.gpu对象，不做任何兼容性封装、不做虚拟化中转

2. 系统启动时检测浏览器WebGPU支持情况，不支持则给出友好提示，不强制降级

3. WebGPU上下文全局唯一，开放给终端WASM模块、AI应用使用，不做权限隔离限制

4. 禁止使用第三方WebGPU框架，原生API直接调用，保证性能无损

## 5\.3 终端与npm兼容规则

1. 仅实现基础npm能力：包安装、脚本执行、依赖加载，不完整兼容原生npm所有命令

2. 所有包安装、文件写入均在浏览器虚拟文件系统内，不接触本地真实磁盘

3. Node运行时基于WASI标准，仅支持纯前端、无系统底层调用的JS包

4. 内置Iris WASM\-LLM预加载逻辑，可通过终端命令直接启动、调用

## 5\.4 数据安全规则

1. 用户API Key、配置参数仅存储在localStorage和IndexedDB，无任何上传逻辑

2. AI对话请求直接从浏览器前端发送至用户配置的接口地址，无任何中转服务

3. 不埋点、不上报用户行为、不收集任何用户数据

4. 提供一键清空所有本地数据的功能，用户可完全清除本地留存内容
